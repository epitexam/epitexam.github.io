Un portfolio, c’est la vitrine pour convaincre des clients et montrer ses skills, mais sans budget, faut être malin. En 2025, un site doit charger en moins d’une seconde, ranker sur Google, et coûter zéro. 

Ce guide compare les outils pour un portfolio rapide, sécurisé, et gratuit, testés sur un mobile entrée / millieu de gamme (Iphone 12). Pour les devs front, freelances, ou designers qui codent, voici comment faire un site qui claque sans se ruiner.
<br/>
## Les bases d’un portfolio
Un portfolio doit être rapide, visible, et tenir dans un budget nul. Le LCP (Largest Contentful Paint) doit être sous 1 seconde pour un score excellent et garder l’audience. Le bundle JavaScript reste sous 5 Ko minifié et gzippé, car un portfolio n’est pas une app lourde. Le build pour 10 pages prend moins de 5 secondes. Le SEO s’appuie sur des balises OpenGraph et schema.org pour briller sur X ou LinkedIn. 

L’accessibilité, c'est non négociable : contraste minimum 4,5:1 pour le texte normal. Pourquoi ? Parce que 27 % des adultes en Europe ont un handicap. Résultat ? Ils sont 12,9 points de pourcentage moins nombreux à utiliser internet tous les jours que les autres.
Côté sécurité : des headers HTTP blindés et zéro faille ouverte.

Quant au style et à la disposition des éléments, l’utilisateur doit comprendre en un coup d’œil qui tu es, ce que tu fais et si tu es disponible. La page d’accueil doit aller droit au but : pas de surcharge d'animation ou d'elements. On garde les fulgurences visuel et la créativité pour les projets personnels. C'est en tout cas ce que je preche.
<br/>
## Astro : Vitesse avec du style
Astro délivre du HTML pur avec du JavaScript uniquement sur les parties interactives, comme un filtre de projets. En 4G throttlée, le LCP descend sous 1 seconde, le bundle JS reste à 3 Ko pour un filtre Svelte, et le build prend environ 2 secondes pour 10 pages. Le déploiement est gratuit sur Cloudflare ou Github Pages avec bande passante illimitée. Exemple de page :

```astro
// src/pages/index.astro
<script>
  import ProjectFilter from '../components/ProjectFilter.svelte';
  import { getCollection } from 'astro:content';
  const projects = await getCollection('projects');
</script>
<main>
  <ProjectFilter client:load />
  {projects.map(p => <article>{p.data.title}</article>)}
</main>
```

Si les configs Vite ou JavaScript te gonflent, Astro peut rebuter, mais pour un site rapide et stylé, c’est du costaud.
<br/>
## Hugo : Rapide comme l’éclair
Hugo, écrit en Go, génère du HTML pur sans JavaScript sauf si tu en ajoutes. En 4G, le LCP tombe sous 1 seconde, le bundle JS reste à zéro par défaut, et le build prend moins d’une seconde pour 10 pages. Le déploiement se fait gratuitement sur GitHub Pages. Exemple :

```html
<!-- layouts/index.html -->
{{ range (where .Site.RegularPages "Section" "projects") }}
  <article>{{ .Title }}</article>
{{ end }}
```

Les templates Go demandent un peu d’apprentissage, mais un thème comme Ananke se configure en dix minutes en passant par git. La sécurité s’ajoute via un fichier `_headers` pour le CSP. Hugo offre la vitesse maximale à budget nul.
<br/>
## Eleventy : Simplicité et efficacité
Eleventy fonctionne en JavaScript avec Markdown ou Nunjucks et livre des performances solides. En 4G, le LCP reste sous 1 seconde avec un filtre statique, zéro JavaScript, et le build prend environ 1 seconde pour 10 pages. Le déploiement est gratuit sur Cloudflare Pages. Exemple :

```
<!-- index.njk -->
{% for project in collections.projects | filterByTag(tag) %}
  <article>{{ project.data.title }}</article>
{% endfor %}
```

La configuration dans `.eleventy.js` reste fluide pour un portfolio simple. La sécurité passe par un fichier `_headers` pour le CSP. Eleventy permet de coder vite et bien sans prise de tête.
<br/>
## Sécu de base : Les pièges à éviter sur un statique
Un portfolio statique est blindé par nature (pas de DB, pas de backend), mais si t'ajoutes des forms ou du JS, tu peux perdre le controle rapidement. Voici ce qu'il faut vérifier: 

- **Configs serveur bancales** : Assure-toi que ton hosting (Cloudflare/Netlify) a des updates auto et pas de ports exposés (genre panels admin oubliés). Un scan rapide avec un tool gratuit comme Nikto te dit si y'a du leak. 

- **Headers HTTP absents** : Sans CSP (bloque les scripts louches) ou HSTS (force HTTPS), t'es vulnérable aux injections CSS/JS ou au sniffing. Ajoute-les via un fichier `_headers` – 5 min chrono. 

- **JS et forms vulnérables** : Si t'as un filtre interactif ou un contact form (Netlify Forms), échappe les inputs pour éviter XSS basique. Exemple pour un form : valide côté serveur avec une regex simple contre < > ;. 

Pour du plus deep (injections, access control), file direct sur OWASP Top 10 – c'est la bible, mais garde ça pour quand ton site grandit.

## Vigilance outils : L'exemple CVE Astro (et pourquoi checker toujours)
Même avec un générateur comme Astro, Hugo ou Eleventy, une faille dans l'outil peut te plomber si t'es pas à jour. Prends la [CVE-2025-61925](https://nvd.nist.gov/vuln/detail/CVE-2025-61925) sur Astro : découverte fin octobre 2025, c'est un spoofing du header X-Forwarded-Host non validé, qui mène à du SSRF (Server-Side Request Forgery) ou manip d'URL (cache poisoning). Impact modéré pour du statique, mais si t'es en SSR mode, ça peut exposer des données internes. 

Le fix ? Un [middleware](https://docs.astro.build/en/guides/middleware/) qui check l'host, comme ça :

```js
// middleware.js (Astro)
export function onRequest({ request }, next) {
  if (!request.headers.get('X-Forwarded-Host')?.includes('ton-site.com')) {
    return new Response('Host invalide', { status: 403 });
  }
  return next();
}
```

Leçon : Avant de déployer, scanne les advisories sur GitHub/NVD pour ton outil (Astro security, Hugo issues). C'est gratuit, ça prend 2 min, et ça évite les surprises. En 2025, avec les updates mensuels, c'est ton job de dev fauché de rester sur le qui-vive.

## Les pièges qui peuvent tout faire capoter
Les images mal optimisées plombent la vitesse : passe en WebP avec lazy loading et dimensions fixes :

```html
<img src="projet.webp" loading="lazy" width="800" height="600" alt="Projet X">
```

Sans OpenGraph, le site reste invisible sur X :

```html
<meta property="og:title" content="Portfolio Perso">
<meta property="og:image" content="/og-image.webp">
```

Teste le contraste avec WAVE et vise au moins 4,5:1 :

```css
body { color: #333; background: #fff; }
```
<br/>
## Hosting : Netlify, Vercel, GitHub Pages, Cloudflare
L’hosting gratuit impacte les perfs et les limites.

Netlify offre 100 Go de bande passante et 300 minutes de build par mois, mais les builds traînent. 

Vercel donne 100 Go avec des builds rapides, mais les cold starts ajoutent du délai sur les fonctions serverless. 

GitHub Pages propose 100 Go sans limite de builds, idéal pour la simplicité. 

Cloudflare Pages livre des builds ultra-rapides avec 500 builds par mois et bande passante illimitée, parfait pour le scaling. Tous gèrent HTTPS et CSP via `_headers`.

Cloudflare domine en performance , GitHub Pages en facilité.
<br/>
## Choisir en 30 secondes
Dev front dans l’écosystème JavaScript ? Eleventy ou Astro sur Cloudflare. Freelance qui veut ranker vite ? Hugo sur GitHub Pages.
<br/>
## Conclusion : Un site pro sans un rond
En 2025, un portfolio doit charger vite, ne rien coûter, et rester blindé. Hugo et Eleventy excellent en vitesse et simplicité, Astro ajoute l’interactivité stylée. Cloudflare Pages règne sur l’hosting gratuit. Code un proto, teste, sécurise, et c’est plié. Un portfolio qui claque, sans un centime.
